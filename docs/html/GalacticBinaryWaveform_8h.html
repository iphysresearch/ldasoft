Galactic binary waveform generator using fast-slow decomposition first described in Cornish and Littenberg, PRD 76, 083006.

Computes the frequency domain TDI response to a circular, slowly evolving, binary with parameters params. The detector geometry is defined in Orbit. The format of the TDI data, either "phase" or "frequency" is specified by format. The TDI response is defined in LISA_tdi() (phase) or LISA_tdi_FF() (frequency).

The TDI response is returned for a single Michelson-like channel, or the two orthogonal channels A and E. The T channel is practically a noise-only channel at typical UCB frequencies.

The array params[] is required to contain parameters: \( f_0T,\cos\theta,\phi,\log\mathcal{A},\cos\iota,\psi,\varphi_0\), where \(\theta\) is the ecliptic co-latitude and \(\phi\) is the ecliptic longitude. Extra parameters for non-zero frequency evolution are optional: \( \dot{f}T^2, \ddot{f}T^3 \).

Parameters
[in]orbitLISA ephemerides
[in]formatTDI format, "phase" or "frequency"
[in]Tobservation time \( T_{\rm obs}\ [{\rm s}]\)
[in]t0start time of observations \( t_0\ [{\rm s}]\)
[in]params[]source parameters
[in]NPnumber of source parameters (7, 8, or 9)
[in]BWsource bandwidth [bins]
[in]NInumber of interferometer channels (1 for X, 2 for A,E)
[out]Xsingle Michelson channel (4-link constellation)
[out]A,Enoise orthogonal TDI channels

aligns generated UCB waveform with Data array

The waveforms generated by galactic_binary() are symmetric about the carrier frequency \(f_0\). To use the template waveform when analyzing the data it must be aligned with the frequency segment. This function computes the bandwidth of the source with parameters Source::params, and computes the offset between the first frequency bin of the template, Source::qmin, and the first frequency bin of the data Data::qmin.

Parameters

[out]Source::BWbandwidth in frequency bins

computes Fisher Information Matrix for UCB waveform parameters Source::params

Computes matrix elements numerically using central differencing \(\Gamma_{ij} = \frac{\partial h}{\partial \theta_i} \frac{\partial h}{\partial \theta_j} \) and stores in Source::fisher_matrix. Matrix eigenvectors and eigenvalues are then computed using matrix_eigenstuff() and stored in Source::fisher_evectr and Source::fisher_evalue, respectively.

[out]Source::fisher_matrix
